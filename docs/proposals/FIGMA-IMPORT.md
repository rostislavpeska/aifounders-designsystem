## 3. The scope→mode ruling (delta analysis, 2026-07-07)

Computed value-level deltas from the manifest (not name counts — real
`ref` differences):

| scope | real deltas | vs |
|---|---|---|
| dark-1 | 30/30 | Light base |
| brand | 26 | Light base |
| support | 7 | Light base |
| light-2 | 2 (`bg`, `perex-border`) | Light base |
| light-3 | 4 (`bg`, `raised`, `perex-border`, `button-tertiary-bg-hover`) | Light base |
| dark-2 | **1** (`bg`→dark-900) | dark-1 |
| dark-3 | 3 (`bg`, `raised`, `button-tertiary-bg-hover`) | dark-1 |

**Option A — all 8 scopes as Semantic modes (RECOMMENDED).** Since the
Schema 2025 raise, Professional allows 10 modes — 8 fits with room to spare.
This is the exact mirror of the CSS scope classes: applying `dark-2` to a
section frame (`setExplicitVariableModeForCollection`) re-resolves every
bound child, including `raised` cards and button knobs, precisely like the
`.section-dark--deep` class does. Nothing is hand-swapped, ever. Cost:
the shade modes repeat ~47 identical aliases each (script-generated, zero
maintenance — the manifest is the source).

**Option B — 4 modes (Light/Dark/Brand/Support) + `surface/*` shade group.**
The pre-raise design, kept for the record: shades become ~10
explicitly-applied variables; a raised card on light-3/dark-3 needs a manual
fill swap. Only relevant if the file must survive on a plan without the
10-mode raise.

The scope axis lives ONLY in Semantic and the brand axis ONLY in Palette —
the web sweep's multi-brand caveat: Figma cannot link same-named modes
across collections, so an axis must never be duplicated into two collections.

## 4. THE AUTO-LAYOUT LAWS (the "clean autolayout" answer)

Distilled from `figma-use` SKILL.md rules 8/12/12a/12b/13/15/16/17 +
`gotchas.md` + `component-patterns.md` + `figma-generate-library`
`component-creation.md`. These are binding for every `use_figma` write:

1. **Every container is auto-layout.** `figma.createAutoLayout(direction,
   props)` is the only container primitive; a bare `createFrame` with
   absolute-positioned children is the #1 tell of a machine-generated mess.
   Absolute x/y exists only at page level (placing top-level frames) and for
   `layoutPositioning: 'ABSOLUTE'` overlays (badges, close buttons, the
   chatbot-clearance ghost) — which must then be sized with `resize()`.
2. **Append first, size after.** `HUG`/`FILL` are value-restricted by
   structural context: `appendChild` into the auto-layout parent, THEN set
   `layoutSizingHorizontal/Vertical`. Fresh unparented nodes reject both.
3. **Two sizing enums, never crossed.** Child: `layoutSizing*` =
   `FIXED|HUG|FILL`. Frame itself: `primaryAxisSizingMode` /
   `counterAxisSizingMode` = `FIXED|AUTO`. (`AUTO` is not a child value;
   `FILL` is not a frame value.)
4. **`resize()` resets sizing modes to FIXED** — always `resize()` first with
   real dimensions, then set `AUTO`/`HUG`. Never pass throwaway 1×1 sizes.
5. **Text law:** default `textAutoResize: 'WIDTH_AND_HEIGHT'` IGNORES `FILL`
   and collapses to a 1-char-per-line thread. Wrapping text =
   `textAutoResize: 'HEIGHT'` + `layoutSizingHorizontal: 'FIXED'` +
   explicit `resize(width, h)`. `HUG` is legal only on auto-layout frames
   and TEXT children.
6. **No `STRETCH`:** `counterAxisAlignItems` has no STRETCH — stretch a child
   with `layoutSizing* = 'FILL'` instead. A FILL child needs a parent with
   committed size (`FIXED` or itself `FILL`), else it collapses.
7. **Fonts before text, always.** `loadFontAsync` → await → mutate. For
   existing text, load the node's CURRENT fonts via
   `getStyledTextSegments(['fontName'])`. Discover exact style strings with
   `listAvailableFontsAsync` (never guess "Black" vs "900").
8. **Every visual property binds to a variable.** Fills/strokes via
   `setBoundVariableForPaint` (returns a NEW paint — reassign!), padding/gap
   via `setBoundVariable('paddingTop'|'itemSpacing'|…)`, radius per-corner
   (`topLeftRadius`…, not `cornerRadius`). A hardcoded hex in a component is
   the Figma equivalent of a lint error. (Empty-fills gotcha: put a
   placeholder SOLID first, then bind.)
9. **Scopes always explicit, never `ALL_SCOPES`:** bg roles →
   `FRAME_FILL,SHAPE_FILL`; text → `TEXT_FILL`; borders → `STROKE_COLOR`;
   spacing → `GAP`; radii → `CORNER_RADIUS`; type → `FONT_*`/`LINE_HEIGHT`.
   Palette primitives → `scopes: []` (invisible in pickers = the
   palette-direct GM law, enforced by Figma itself).
10. **`codeSyntax` carries the EXACT CSS custom property** —
    `setVariableCodeSyntax('WEB', 'var(--text-secondary)')`, taken verbatim
    from the manifest, never derived from the Figma name. This is what makes
    Dev Mode print our tokens instead of hex.
11. **Variant sets:** component properties (`addComponentProperty` — capture
    the returned key, it's suffixed) and explicit
    `setExplicitVariableModeForCollection` per variant BEFORE
    `combineAsVariants`; after combining, children stack at (0,0) — grid them
    (columns = State, rows = Size×Style) and `resizeWithoutConstraints` from
    actual child bounds. **≤30 combinations per set**; icons and slots go
    through `INSTANCE_SWAP`/`SLOT` properties, never variant axes.
12. **Session mechanics:** page context resets every `use_figma` call
    (`setCurrentPageAsync`, once per call); batch independent awaits with
    `Promise.all`; return ALL created/mutated node IDs from every script;
    place top-level nodes away from (0,0).

Community doctrine on top of the API rules (web sweep, convergent across
UIPrep / goodpractices.design / LogRocket / Steve Kinney — sources in §11):

13. **One dimension owner per axis.** Parent-hug XOR child-fill — a FILL
    child silently flips its parent's axis to Fixed, so decide who owns each
    dimension and never let them fight. Component roots hug both axes; text
    leaves hug; flexible children fill.
14. **No wrapper without a job.** A nested frame must bring its own
    direction change, gap/padding, or fill — otherwise it's wrapper-div-itis
    and pollutes handoff (layers ≈ DOM). Flatten it.
15. **Min/max over Fixed for resilience** — fill-with-max-width /
    hug-with-min-width instead of hardcoded Fixed; baseline alignment for
    icon+label rows; strokes don't affect layout size by default (mismatches
    CSS `border` — set strokes-included-in-layout deliberately; our fields
    carry 2px borders, so this is live for Input/Newsletter capture).
16. **Name at creation.** "Frame 47 / Rectangle 203" is THE
    machine-generated-file smell; every node gets its semantic name in the
    same script that creates it (ours come from the ledger class names).

## 5. Typography policy

- 20 text styles as `Category/Name` (`Display/Hero`, `Heading/XL`,
  `Body/MD`…), values resolved from Type Primitives.
- **Clamp resolution:** the 3 fluid sizes materialize per Breakpoint mode —
  Desktop = clamp max at 1440 (96/80/64), Mobile = clamp floor at 390
  (48/40/36) — the same two viewports `shots.mjs` renders. If text-property
  variable binding works in the import session (`setBoundVariable('fontSize')`
  — the skill docs are ambivalent; verify live), styles flip with the mode;
  otherwise materialize **two style sets** `…/Desktop` + `…/Mobile`.
- **brandType** (heading-md font/weight, lead leading, …) rides Palette's
  brand modes as STRING/FLOAT aliases into Type Primitives — so AIF text
  styles resolve Inter/extrabold where AIG resolves Space-Grotesk-display/black,
  from the same style definition.
- `heading-lg-size` is a per-brand clamp (44 vs 36 desktop max) → FLOAT in
  Palette with per-brand-per-breakpoint values via the Breakpoint modes of
  Type Primitives (or two FLOATs if binding fights us — decided live).

## 6. Component import order (the 40 rows, dependency-tiered)

Per `figma-generate-library` Phase 3 (atoms before molecules; one page per
component; `_`-prefixed internals hidden from Assets):

- **Foundations (not component sets):** Text styles · Text elements ·
  Numbered headings · Prose/article layout · Global link system · Card link
  utilities · Section contexts (surfaces) → Foundations pages: color swatch
  boards (all 4 semantic modes × 2 brands), type specimens, spacing bars,
  surface/scope documentation.
- **Icons:** 117 `Icon/<name>` components (bulk SVG import, one page).
- **Tier 0 atoms:** Button · Badge · Avatar · Input (+textarea) · Select ·
  Checkbox · Radio · Segmented control.
- **Tier 1 molecules:** Newsletter capture (the `.input-pair` conjoin law —
  row join via auto-layout gap 0 + stretch, consent-note slot) · Datepicker ·
  Consent (GDPR) · Info box · Info bar · Breadcrumb · Pagination · Nav tabs ·
  Accordion · Data table · Record list · Blurb · File dropzone · Sticky bar.
- **Tier 2 organisms:** Preview card · Reference card · Persona card ·
  Course card · Engagement · Comments · Modal · Blurb + stack grid (pizza
  law: divided grid = interior-only separators — in Figma, the dividers are
  1px stroke-only frames INSIDE the grid, never edge strokes on cells) ·
  Header · Footer · Forms composition (pattern page).

Each component: base component with all children + variable bindings →
variants (states/sizes within the ≤30 cap) → component properties (TEXT
label, BOOLEAN slots like consent-note/eyebrow, INSTANCE_SWAP icon) →
documentation frame (intent sentence from its `docs/components/*.md`) →
`get_screenshot` validation against the styleguide specimen.

## 7. Session protocol (idempotency + recovery)

From `figma-generate-library/references/error-recovery.md`, binding for us:

- **Tag every created node** `setSharedPluginData('dsb', 'key', …)` +
  `run_id` — cleanup and resume run off plugin data, never name matching.
- **Check-before-create** by `dsb key` (collections, pages, component sets)
  — every step idempotent, safe to re-run.
- **State ledger → `assets/figma-map.json` (committed):** name → node id,
  variable ids, file key, run id. This file is ALSO the Code Connect /
  rows backfill source (§8) — one artifact, two consumers.
- On failure: STOP → `get_metadata`/`get_screenshot` → fix → re-run (calls
  are atomic). On context loss: rehydrate by scanning `dsb` tags.
- File creation: `figma-create-new-file` skill → `create_new_file` in the
  "My Projects" team, from the cover-page template; pages `Cover`, `---`,
  `Foundations`, `Icons`, `---`, then component pages in §6 order.

  **Organization/Enterprise** — it is NOT available on Professional. The
  MCP mapping calls will be permission-refused on "My Projects". So:

  - `code_connect` on the rows stays null with reason recorded
    (`"deferred: figma plan"`). The day the org upgrades, the path is
    first-party: Code Connect **HTML** accepts any markup —
    `figma.enum` variant→class mapping + `figma.className` were built for
    CSS-class systems exactly like ours; one `*.figma.ts` per row,
    `source` = the row's `git_path`, published via `figma connect publish`.
- Variables still round-trip TODAY through `codeSyntax` (§4 law 10) — Dev
  Mode emits `var(--…)` on every inspected property regardless of plan.
  That carries most of the designer→developer token fidelity on its own.

## 11. Web-sweep findings (2026-07-07)

What the web added beyond the bundled skills, with confidence:

- **Mode limits raised at Schema 2025** (Oct 28 2025, Figma staff in the
  "[LAUNCHED] more than 4 variable modes" thread): Pro 4→**10**, Org 4→20,
  Enterprise 40; Starter cannot create modes. HIGH. → §3 Option A viable.
- **Cross-collection aliasing + independent per-collection mode resolution:
  settled.** Modes apply per collection per frame; unset collections resolve
  via Auto up the tree. Caveat: no mechanism links same-named modes across
  collections → **one axis per collection, never duplicated**. HIGH.
- **Extended Collections** (parent + per-brand children, Schema 2025):
  Enterprise-only; migrating drops applied modes. Not for us. HIGH.
- **Code Connect: Organization/Enterprise only** — the §8 gate. Its HTML
  integration is first-party and fits CSS-class systems (`figma.enum`,
  `figma.className`, any markup; snippets static, no logic). HIGH.
- **Clean-auto-layout community doctrine** → §4 laws 13–16 (dimension owner,
  wrapper justification, min/max resilience, baseline, stroke-vs-border
  caveat, naming). MEDIUM-HIGH, convergent (UIPrep, goodpractices.design,
  LogRocket MCP guide, Steve Kinney).
- **Messy-file tells** (what reviewers flag in generated files): "Frame 47"
  names, hardcoded values instead of bindings, absolute positioning as
  layout, detached instances, purposeless nesting, sibling components with
  inconsistent hug/fill. Our gates: §4 laws + §7 screenshot validation.
- Unverified residue (recorded honestly): exact Enterprise cap post-raise
